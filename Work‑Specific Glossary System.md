Proposal: Improving Novel Posting Platforms through Work-Specific Glossaries
(Reducing user drop-off and increasing on-site engagement and revenue retention)
________________________________________
1. Purpose
To prevent misinterpretation of work-specific terminology and reduce user drop-off caused by external searches, this proposal introduces an on-site glossary system that allows readers to resolve terminology without leaving the platform.
This system directly improves user retention, increases session duration, and reduces revenue leakage caused by off-site navigation.
________________________________________
2. Current Issues
Terminology used within novels often differs from general usage, and identical words may carry different meanings across works.
Examples:
•	Potion: rare, high-value healing item → common household medicine
•	Ray gun: special military equipment → standard infantry weapon
As a result, readers frequently leave the platform to search for meanings on external sites.
This leads to:
•	Disruption of reading flow
•	Premature user exit (drop-off)
•	Exposure to spoilers
•	Misinformation from unreliable sources
•	Loss of monetization opportunities
________________________________________
3. Solution: On-Site Glossary System
3-1. Basic Glossary (Background Dictionary)
By extending the existing text-selection UI, users can tap a term to display a floating panel with a brief definition.
Features:
•	Instant inline meaning display
•	Auto-dismiss on scroll
•	Expandable detailed view
Impact:
•	Eliminates reading interruption
•	Improves comprehension without leaving the platform
________________________________________
3-2. Work-Specific Glossary (Author-Editable, Optional)
Since terminology varies by work, authors can optionally define and manage their own glossary entries.
Example: Same name, different meanings
•	Author A, Work 001: Aria (rural origin)
•	Author A, Work 002: Aria (secretary)
•	Author B, Work 003: Aria (soldier)
The system identifies the active work and automatically displays the correct definition.
Impact:
•	Prevents confusion across works
•	Enables precise understanding with minimal user effort
•	Allows data collection on commonly misunderstood terms
________________________________________
3-3. Dual Definition Display (Work-Specific + General Meaning)
For ambiguous terms, both meanings are displayed:
•	Work-specific definition (author priority)
•	General dictionary definition
Impact:
•	Prevents misinterpretation
•	Supports both casual and advanced readers
________________________________________
3-4. Progressive Disclosure (Chapter-Based Unlocking)
Definitions update dynamically based on reading progress.
Example: Character relationship
•	Chapter 1–: Rival
•	Chapter 5–: Shared past
•	Chapter 9–: Reconciliation
Impact:
•	Prevents spoilers
•	Aligns information with narrative intent
________________________________________
4. Business Impact
This system provides direct and measurable value:
•	Reduced user drop-off rate
•	Increased session duration
•	Improved ad retention and visibility
•	Higher content completion rates
•	Enhanced user satisfaction and platform loyalty
________________________________________
5. Future Development
Traditionally, printed novels and manga included static glossaries or character charts.
This proposal evolves that concept into a dynamic, scalable linguistic infrastructure (“lexical assets”).
Organized terminology can support:
•	Consistent terminology in print publications
•	Source material for official guides and settings
•	Stable translation for international releases
•	Expandable reference data for fan engagement
________________________________________
Conclusion
By keeping readers engaged within the platform and reducing reliance on external searches, this system directly improves retention, monetization efficiency, and long-term content value.
This proposal is submitted from a user perspective with the goal of enhancing both user experience and platform performance.
